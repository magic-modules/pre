@magic-modules/pre

this is the @magic-modules Pre component. syntax highlighting for javascript

installation:

installation is done using npm. for now, all magic modules are living on github and not on npm.

note the missing @ before magic-modules.

this is how we install npm modules from github.

npm install magic-modules/pre

require:

first add the component to the assets

// assets/index.jsmodule.exports = {  // ..other exports  Pre: require('@magic-modules/pre'),}

usage:

in a page/component, just pass some string that looks like js

module.exports = {  View: () => Pre('const js = true'),}

renders

const js = true

colors

Pre supports two color themes

fixed color:

you can give a fixed color to a Pre by passing a second argument with the color.

as you can see below, this also removes the theme choose buttons

dark

Pre('content', 'dark')

light

Pre('content', 'light')

syntax

comments

Pre can handle single line comments starting with: //.Multiline comments are on the todo list.

const v = 't' // single line comment

urls

Pre can handle urls even though they look like comments. Those urls will even turn into Link elements. Urls are identified by containing the three characters ://

https://jaeh.at

Pre even handles links in comments and strings

before comment // comment http://link.in.comment

emails

Pre finds emails in your text. emails can either start with mailto: or must include at least one @ and at least one .yes, strictly speaking, name@host is a valid email, but how often have you seen that?

if you actually do need support for name@local, please file an issue.

both examples render the same

Pre('mail@jaeh.at')
Pre('test.mail@jaeh.at')
Link({ to: 'mail@jaeh.at' }, 'mail@jaeh.at')

result

mail@jaeh.at

link and comment

const v = 't' // comment https://jaeh.at// and in a comment starting the line https://wizardsatwork.at
